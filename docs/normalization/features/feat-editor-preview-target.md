# Фича: превью в редакторе (целевое состояние)

**Родитель:** целевое состояние. Роадмап **п.3** ⏳.

---

## Описание

Режим визуализации в **normalization-tool** должен использовать **тот же** контур рендера, что **transpile-viewer** / внешний потребитель: **`FormRenderer`** и согласованная обвязка, без расхождения семантики **tool vs viewer**.

---

## Требования (цель)

- **`editor.preview.embed-viewer`:** в режиме превью — встраивание движка **transpile-viewer** (или общей библиотеки на его базе), вместо или поверх **`FormPreview`** (UX — при внедрении).
- **`editor.preview.same-import-graph`:** один контур импорта рендера, те же правила **`configHook`** и песочницы модулей.
- **`editor.preview.scenario-hook` (черновик):** опциональный модуль в духе **`handlers/preview-scenario.ts`**, вызываемый **только в превью**, возвращающий моки / патч **`state`**; точное имя поля и сигнатура — при реализации.

---

## Открытые вопросы (продукт)

- Синхронизация **`rows`/`cols`** у Grid/Table в DSL с возвратом хука.
- Единый контракт **тулбара** и **`rowSelection`** у Table (DSL vs Hexa из хука).
