# Definition of done

При работе с компонентами (создание/правки) рекомендуется добавить в описание Pull Request'а чек-лист с критериями законченности и корректности вносимых изменений.
Так называемый Definition Of Done.

Отметить галочками каждый пункт и удостовериться, что при разработке ничего не забыто.
Список не строгий и нужен исключительно для самоконтроля.

---

### Создание компонента:

- [x] Создана story с примерами использования компонента
- [x] Примеры использования достаточно наглядны и всесторонни
- [x] Поддержана темная тема и добавлен ее пример в story
- [x] Компонент реэкспортирован в глобальном index файле
- [x] Поддержаны HTML-аттрибуты для тестирования (если нужны) и возможность их определения извне (через props)
- [x] Текстовые элементы (если есть) не захардкожены и есть возможность их определения извне (через props)

---

### Добавление функциональности к существующему компоненту:

- [x] Добавлены примеры использования в story
- [x] Все предыдущие story этого компонента проверены и работают корректно
- [x] Темная тема работает корректно, в том числе для нового функционала
- [x] Поддержаны HTML-аттрибуты для тестирования (если нужны) и возможность их определения извне (через props)
- [x] Текстовые элементы (если есть) не захардкожены и есть возможность их определения извне (через props)

---

### Исправление/изменение функциональности существующего компонента:

- [x] Все story этого компонента проверены и работают корректно
- [x] Темная тема работает корректно


### Разработка документации

Docs каждого компонента должен быть обернут с помощью hoс `WithMeta`, который в свою очередь обогащает документацию мета-информацией.
