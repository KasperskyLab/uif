Используйте компонент в ограниченных по размеру областях интерфейса, где контент должен оставаться доступным без перестройки всего экрана: в боковых панелях, длинных списках, содержимом модальных окон, карточках и других контейнерах с фиксированной или предсказуемой высотой. Если блок может естественно расти вместе с контентом, отдельный прокручиваемый контейнер обычно не нужен. Не превращайте в **Scrollbar** всю страницу: для общего скролла экрана оставляйте нативное поведение макета."
