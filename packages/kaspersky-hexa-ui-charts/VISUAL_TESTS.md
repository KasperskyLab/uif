# Скриншотные тесты

В `hexa-ui-charts` есть скриншотные тесты для проверки визуальных изменений компонентов.

Тесты запускаются через докер, чтобы результаты были одинаковыми на разных системах и не зависели окружения.

## Требования

Должен быть установлен докер.

## Запуск

```bash
npm run test:visual
```

Запускает скриншотные тесты.

```bash
npm run test:visual:update
```

Запускает тесты и обновляет **все** скриншоты.

> Перед запуском `npm run test:visual:update` обязательно запустите `npm run test:visual` и проверьте, что различаются только те стори, которые действительно нужно обновить.

## Принцип работы

1. В контейнере устанавливаются зависимости через `npm ci --prefer-offline`. (Если зависимости устанавливались ранее, они возьмутся из кэша, благодаря разделу описанному в композе)
2. Собирается сторибук.
3. Тест `tests/visual/visual.test.ts` читает собранный сторибук из `storybook-static`.
4. Playwright открывает каждую сторю, делает скриншот и сравнивает его с сохрненным скриншотом.

Если скриншоты отличаются, тест падает, а информация о различиях сохраняется в `test-results`.

## Как исключить сторю из скриншотных тестов

Если стори не подходит для скриншотного тестирования, добавляется тег:

```ts
export const RandomValues = {
  tags: ['skip-visual'],
};
```

Такая стори не будет проверяться в скриншотных тестах.

## Задержка перед скриншотом

Для сторей с анимациями можно задать задержку перед созданием скриншота:

```ts
export const Speedometer = {
  parameters: {
    visual: {
      delayMs: 2500,
    },
  },
};
```

Перед тем как сделать скриншот воркер подождет указанное количество миллисикунд.

По возможности, лучше не использовать, так как использование увеличивает время проверки стори.
